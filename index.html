Modul 03 — Iterasi & Koleksi

Looping
& Array

Loop mengulang eksekusi kode. Array menyimpan banyak nilai bertipe sama dalam satu variabel terindeks mulai dari 0.

Tiga jenis loop

Loop.java
// 1. FOR — jumlah iterasi sudah diketahui
for (int i = 0; i < 5; i++) {
    System.out.print(i + " "); // 0 1 2 3 4
}

// 2. WHILE — periksa kondisi SEBELUM iterasi
int x = 1;
while (x <= 5) {
    System.out.print(x + " ");
    x++;
}

// 3. DO-WHILE — eksekusi MINIMAL sekali, cek kondisi SESUDAH
int n = 10;
do {
    System.out.println("Pasti jalan: " + n);
    n++;
} while (n < 5); // kondisi false, tapi sudah jalan sekali

// BREAK & CONTINUE
for (int i = 0; i < 10; i++) {
    if (i == 3) continue; // lewati i=3
    if (i == 7) break;    // berhenti di i=7
    System.out.print(i + " "); // 0 1 2 4 5 6
}

Array 1D & 2D

Array.java
// Deklarasi Array 1D
int[] nilai = {85, 90, 78, 92, 88};
System.out.println(nilai[0]);          // 85 (index pertama)
System.out.println(nilai.length);      // 5 (jumlah elemen)

// For-Each (enhanced for) — lebih ringkas
for (int n : nilai) {
    System.out.print(n + " ");
}

// Array 2D (matriks baris x kolom)
int[][] matriks = {
    {1, 2, 3},   // baris 0
    {4, 5, 6},   // baris 1
    {7, 8, 9}    // baris 2
};
System.out.println(matriks[1][2]);    // 6 (baris 1, kolom 2)

// Iterasi matriks dengan nested loop
for (int i = 0; i < matriks.length; i++) {
    for (int j = 0; j < matriks[i].length; j++) {
        System.out.print(matriks[i][j] + "\t");
    }
    System.out.println();
}

Modul 04 — OOP Inti

OOP Paradigma
Object & Class

OOP mengorganisir kode menjadi objek yang punya atribut (data) dan method (perilaku). Class adalah blueprint, Object adalah instansinya.

Class

Blueprint / cetakan yang mendefinisikan struktur dan perilaku objek

Object

Instance nyata dari class, dibuat dengan keyword new

Attribute

Variabel yang menyimpan data / state dari sebuah objek (field)

Constructor

Method khusus dipanggil saat objek dibuat, nama = nama class

Membuat Class & Object

Mahasiswa.java
public class Mahasiswa {

    // ── ATRIBUT ──────────────────────────────
    String nama;
    int    nim;
    double ipk;
    String jurusan;

    // ── CONSTRUCTOR ──────────────────────────
    public Mahasiswa(String nama, int nim, String jurusan) {
        this.nama    = nama;    // this = objek ini sendiri
        this.nim     = nim;
        this.jurusan = jurusan;
        this.ipk     = 0.0;
    }

    // ── METHOD ───────────────────────────────
    public void perkenalan() {
        System.out.println("Nama    : " + nama);
        System.out.println("NIM     : " + nim);
        System.out.println("Jurusan : " + jurusan);
    }

    public void setIPK(double ipk) {
        this.ipk = ipk;
    }

    public String getStatus() {
        return (ipk >= 3.5) ? "Cumlaude" : "Reguler";
    }
}

// ── PENGGUNAAN (di class Main) ────────────
Mahasiswa mhs1 = new Mahasiswa("Budi", 12345, "Informatika");
Mahasiswa mhs2 = new Mahasiswa("Sari", 67890, "Sistem Informasi");

mhs1.perkenalan();
mhs1.setIPK(3.8);
System.out.println(mhs1.getStatus()); // Cumlaude
Apa fungsi keyword this di dalam sebuah class?

Encapsulation menyembunyikan data internal objek dari akses luar. Data diproteksi dengan private dan dikontrol via getter/setter.

Tabel Access Modifier

Modifier Class Sendiri Package Sama Subclass Beda Package

(default) ✓ Ya ✓ Ya ✗ Tidak ✗ Tidak

Implementasi Encapsulation

RekeningBank.java
public class RekeningBank {

    // Private — tidak bisa diakses langsung dari luar!
    private double saldo;
    private String noRekening;
    private String namaPemilik;

    // Constructor
    public RekeningBank(String noRek, String nama) {
        this.noRekening  = noRek;
        this.namaPemilik = nama;
        this.saldo       = 0;
    }

    // GETTER — membaca nilai private
    public double getSaldo()       { return saldo; }
    public String  getNoRekening() { return noRekening; }
    public String  getNamaPemilik(){ return namaPemilik; }

    // SETTER dengan VALIDASI — kontrol perubahan data
    public void setor(double jumlah) {
        if (jumlah > 0) {
            saldo += jumlah;
            System.out.println("Setor Rp" + jumlah + " berhasil");
        } else {
            System.out.println("Error: jumlah harus positif");
        }
    }

    public void tarik(double jumlah) {
        if (jumlah > 0 && jumlah <= saldo) {
            saldo -= jumlah;
        } else {
            System.out.println("Saldo tidak cukup!");
        }
    }
}

// Penggunaan
RekeningBank rek = new RekeningBank("BCA-001", "Budi");
rek.setor(1000000);
rek.tarik(250000);
System.out.println(rek.getSaldo()); // 750000.0
// rek.saldo = 9999;  ← COMPILE ERROR! Cannot access private field

Inheritance
(Pewarisan)

Class anak mewarisi atribut dan method dari class induk menggunakan keyword extends. Mendukung prinsip reuse kode.

Hierarki Pewarisan

Hewan
nama | makan() | bernafas()
↓ extends
Kucing
+ meong() | + cakar()
Anjing
+ gonggong() | + gigit()

Implementasi Inheritance

Hewan.java + Kucing.java
// ── CLASS INDUK (Parent / Superclass) ────
public class Hewan {
    protected String nama; // protected: bisa diakses subclass
    protected int    umur;

    public Hewan(String nama, int umur) {
        this.nama = nama;
        this.umur = umur;
    }

    public void makan() {
        System.out.println(nama + " sedang makan");
    }

    public void info() {
        System.out.println("Nama: " + nama + ", Umur: " + umur);
    }
}

// ── CLASS ANAK (Child / Subclass) ────────
public class Kucing extends Hewan {
    private String warna;

    // Constructor anak memanggil super() untuk init induk
    public Kucing(String nama, int umur, String warna) {
        super(nama, umur); // panggil constructor Hewan
        this.warna = warna;
    }

    // Method tambahan milik Kucing
    public void meong() {
        System.out.println(nama + ": Meong! Meong!");
    }
}

// ── PENGGUNAAN ────────────────────────────
Kucing kitty = new Kucing("Kitty", 2, "Putih");
kitty.info();    // diwarisi dari Hewan
kitty.makan();   // diwarisi dari Hewan
kitty.meong();   // milik Kucing sendiri

Modul 07 — Pilar OOP #3

Polymorphism

Satu interface, banyak bentuk. Objek berbeda bisa merespons method yang sama dengan cara masing-masing.

Subclass mendefinisikan ulang method dari superclass. Java memanggil versi subclass saat runtime.

Override.java
class Bentuk {
    public void gambar() {
        System.out.println("Menggambar bentuk generik");
    }
    public double hitungLuas() { return 0; }
}

class Lingkaran extends Bentuk {
    private double radius;
    public Lingkaran(double r) { this.radius = r; }

    @Override
    public void gambar() {
        System.out.println("○ Menggambar lingkaran");
    }
    @Override
    public double hitungLuas() {
        return Math.PI * radius * radius;
    }
}

class Persegi extends Bentuk {
    private double sisi;
    public Persegi(double s) { this.sisi = s; }

    @Override
    public void gambar() {
        System.out.println("□ Menggambar persegi");
    }
    @Override
    public double hitungLuas() { return sisi * sisi; }
}

Method dengan nama sama tapi daftar parameter berbeda dalam satu class. Compiler memilih method yang tepat berdasarkan argumen.

Overload.java
class Kalkulator {

    // Overload 1: dua int
    public int hitung(int a, int b) {
        return a + b;
    }

    // Overload 2: dua double
    public double hitung(double a, double b) {
        return a + b;
    }

    // Overload 3: tiga int
    public int hitung(int a, int b, int c) {
        return a + b + c;
    }

    // Overload 4: String concatenation
    public String hitung(String a, String b) {
        return a + b;
    }
}

// Java otomatis pilih versi yang sesuai
Kalkulator k = new Kalkulator();
k.hitung(1, 2);            // → int: 3
k.hitung(1.5, 2.5);       // → double: 4.0
k.hitung(1, 2, 3);         // → int: 6
k.hitung("Java", " OOP"); // → String: "Java OOP"

Objek anak disimpan dalam referensi tipe induk. Method yang dieksekusi ditentukan tipe objek asli saat runtime (dynamic dispatch).

Upcasting.java
// Upcasting: referensi Bentuk, objek Lingkaran/Persegi
Bentuk b1 = new Lingkaran(7);
Bentuk b2 = new Persegi(5);

b1.gambar();       // "○ Menggambar lingkaran"
b2.gambar();       // "□ Menggambar persegi"

// Manfaat nyata: array of Bentuk (polimorfik)
Bentuk[] shapes = {
    new Lingkaran(5),
    new Persegi(4),
    new Lingkaran(3)
};

// Satu loop, tiga bentuk berbeda — inilah kekuatan polymorphism!
for (Bentuk s : shapes) {
    s.gambar();
    System.out.printf("Luas: %.2f%n", s.hitungLuas());
}

// Downcasting — kembalikan ke tipe asli jika perlu
if (b1 instanceof Lingkaran) {
    Lingkaran l = (Lingkaran) b1; // downcasting aman
}
